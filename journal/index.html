Best Practice for Form Design

Afrah Said

Jan/26/2023

This week's article Best Practice for Form Design delved into the topic of forms, specifically the methods used to create a good form design. Just like websites, forms have their own set of practices and patterns. These patterns help guide users through the forms as smoothly as possible. These patterns include single columns, input labels, progress bars, required fields, input requirements, hierarchy of information needed, error feedbacks, and color contrast, just to name a few. A lot of these patterns are implemented to help users better understand each field and the content needed. By making these forms easy to navigate and understand, designers can increase users' retention and likelihood of completing the form.


After analyzing multiple forms, I have realized that they all incorporate many of the practices mentioned in the article. For example, Discord's registration form has input titles, placeholder text, contrasting colors, and provides error feedback messages when an input does not fulfill a requirement. Their login form also uses asterisks to indicate which inputs need to be completed. Similarly, when creating a form in Google Forms, it is structured in the same format as the article describes, with columns, an option for creating a required field, customizable title fields, and a description option where examples and input requirements can be added.
